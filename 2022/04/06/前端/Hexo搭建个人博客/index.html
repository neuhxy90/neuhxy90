Hexo搭建个人博客

发表于 2022-04-06 更新于 2022-09-08 分类于 前端

Hexo 介绍

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

以前写博客大多基于 CKeditor 之类的富文本编辑器，熟悉 Markdown 之后发现 Typora 的体验要更好，于是研究了一下怎么写 markdown, 然后一键推送到服务器形成个人博客，查阅了一些资料，最终发现 Hexo 基本满足需求，所以整理了以下操作记录。

Hexo 部署

常见的博客页面服务提供商可以选择github或者 gitee，github有时候会访问受限，gitee是国内的服务，需要个人身份证正反面+手持身份证照片审核，大约需要一个工作日。github支持一键部署，gitee需要进入pages服务页面手动点更新。

下面以Github为例，登录Github，创建一个仓库，权限为Public即可，命名为自己的用户名.github.io，然后在代码仓库上方的菜单中找到 settings，进入后找到 General 配置下方的 Code and automation -> Pages，开启Pages服务。

接下来按提示操作即可。

部署到github需要安装插件：命令为npm install hexo-deployer-git --save

然后修改根目录下的全局配置： _config.yml

  1. 第14行左右，配置github提供的博客地址
  1. 第104行左右，配置git仓库地址。

创建公钥：

执行完会在自己的C:/Users/NAME/.ssh路径下生成公钥，打开id_rsa.pub，全选复制，

进入Github官网，点击右上方的头像，再按settings进入设置，在Access里点击SSH and GPG keys，点击New SSH key创建，title输入邮箱，key里面粘贴刚刚打开id_rsa.pub里复制的的内容，再点Add SSH key保存。
