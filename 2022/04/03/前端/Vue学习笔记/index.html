Vue 介绍

截止目前，vue 3.x已经是官方定义的默认版本了。参考资料：https://v3.cn.vuejs.org/

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

看官方的介绍，什么叫渐进式框架？是一种比较友好的开发约束，也就是你的开发中使用vue可以满足最小化需求，按需引入，没有强制。举个例子，开发一个简单的应用，那么用vue本身的库就够用了，你需要多个页面跳转，那你再加一个vue-router，你需要数据交互，那么调用store，需要多页面组件共享数据状态同步，那么调用vuex就行了，vue没有强制你开发一个简单的单页面demo应用都必须引入所有库，各个附加库的职责划分很细致，组合也很灵活。

Vue Router

参考资料：https://router.vuejs.org/zh/introduction.html

基本概念

官方定义：

Vue Router 是 Vue.js 的官方路由。它与 Vue.js 核心深度集成，让用 Vue.js 构建单页应用变得轻而易举。功能包括：

  • 嵌套路由映射
  • 动态路由选择
  • 模块化、基于组件的路由配置
  • 路由参数、查询、通配符
  • 展示由 Vue.js 的过渡系统提供的过渡效果
  • 细致的导航控制
  • 自动激活 CSS 类的链接
  • HTML5 history 模式或 hash 模式
  • 可定制的滚动行为
  • URL 的正确编码

解释一下Vue Router就是浏览器访问的url与对应的显示页面的管理工具，因为vue的设计思想是单页面应用，也就是说网站的所有内容都通过一个单独的页面显示，需要用到哪个页面的内容了就替换当前页面的内容。

注意：

路由和菜单的概念很像，但是路由是用来管理页面请求的url对应的vue页面文件的，而菜单是在页面显示的一个个链接与对应的url的。

特点：

在html中，常用<a>处理页面跳转，点击后浏览器会重新加载新页面，对用户的体验不太好，在vue-router中使用router-link和router-view来处理。

  • router-link：相当于要点击的超链接

  • router-view：相当于把超链接对应的页面具体显示在当前页面的那个区域

Vue 组件间传值

Vue的模式和常规的div+css纯前端有些不同，在Vue的开发习惯里，为了让代码复用，提出了组件的概念，这个组件相当于一部分div+css+js开发的页面功能单元，放在特定的目录下（通常为src/components里），然后有专门的页面文件夹，如views里调用这个组件，再根据特定的需求显示出来。

比如，我们开发了一个卡片样式的组件，在不同的页面中显示的大小不同，即父类组件中定义可以动态调整大小的参数，比如height，那么在views里调用的时候，传值给父类组件即可动态地显示页面的大小。就需要用到props关键字。

参考资料：https://v3.cn.vuejs.org/api/options-data.html#props

Vuex

参考资料：https://vuex.vuejs.org/zh/

官方定义：

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式 + 库。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

按照官方的解释，vuex是为了解决多个页面共享组件状态管理的。
